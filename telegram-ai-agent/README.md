# Telegram AI Agent Manager для заявок по сложному имуществу

Минимальный рабочий проект Telegram-бота, который находится в группе, видит входящие сообщения, определяет заявки, задает уточняющие вопросы, формирует карточку лида и сохраняет данные в Google Sheets.

## Что делает бот

1. Читает сообщения в Telegram-группе.
2. Определяет, похоже ли сообщение на заявку.
3. Если это заявка — запускает AI-квалификацию.
4. Задает клиенту короткие уточняющие вопросы.
5. Формирует внутреннюю карточку заявки.
6. Отправляет карточку в группу или отдельный служебный чат.
7. Сохраняет заявку в Google Sheets.

## Стек

- Node.js 20+
- Telegraf
- OpenAI API
- Google Sheets API
- dotenv

## 1. Создать Telegram-бота

1. Откройте Telegram.
2. Найдите `@BotFather`.
3. Выполните `/newbot`.
4. Получите токен бота.
5. Добавьте бота в вашу группу.
6. Чтобы бот видел все сообщения группы, отключите privacy mode:
   - `/setprivacy`
   - выбрать своего бота
   - `Disable`

Важно: если privacy mode включен, бот будет видеть не все сообщения группы, а только команды, упоминания и ответы на свои сообщения.

## 2. Подготовить Google Sheets

1. Создайте Google Sheet.
2. Создайте вкладку `Leads`.
3. Создайте Service Account в Google Cloud.
4. Включите Google Sheets API.
5. Скопируйте email service account.
6. Дайте этому email доступ на редактирование к вашей таблице.
7. Возьмите:
   - `GOOGLE_SHEET_ID` из URL таблицы;
   - `GOOGLE_SERVICE_ACCOUNT_EMAIL`;
   - `GOOGLE_PRIVATE_KEY`.

Если Google Sheets не настроены, бот все равно будет работать, но не будет сохранять заявки в таблицу.

## 3. Настройка `.env`

Скопируйте пример:

```bash
cp .env.example .env
```

Заполните:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_telegram_бота
MANAGER_CHAT_ID=
OPENAI_API_KEY=ваш_openai_api_key
OPENAI_MODEL=gpt-4.1-mini
GOOGLE_SHEET_ID=id_вашей_google_таблицы
GOOGLE_SHEET_TAB=Leads
GOOGLE_SERVICE_ACCOUNT_EMAIL=email_service_account
GOOGLE_PRIVATE_KEY=private_key_service_account
COMPANY_MANAGER_NAME=Андрей
```

`MANAGER_CHAT_ID` можно оставить пустым. Тогда карточки будут приходить в ту же группу.

Чтобы узнать `MANAGER_CHAT_ID`, временно добавьте бота в нужный чат, напишите `/ping`, посмотрите логи или используйте Telegram getUpdates-инструмент.

## 4. Установка и запуск

```bash
npm install
npm run start
```

Для разработки:

```bash
npm run dev
```

## 5. Проверка

Напишите в группе пример заявки:

```text
Нужно продать производственную базу в Московской области. Продаем больше года, звонков почти нет. Цена 85 млн.
```

Ожидаемый результат:

1. Бот задаст уточняющие вопросы.
2. Бот сформирует карточку заявки.
3. Карточка появится в группе или служебном чате.
4. Строка появится в Google Sheets.

## 6. Логика статусов

- `новая` — заявка только поступила.
- `требуется уточнение` — данных недостаточно.
- `целевая` — актив подходит компании.
- `нецелевая` — заявка не соответствует профилю.
- `передана Андрею` — можно брать в работу.
- `закрыта` — заявка завершена.

## 7. Деплой

Подойдут:

- Render Worker
- Railway
- VPS
- Docker/VPS

Для Netlify Functions polling-бот подходит плохо, потому что нужен постоянный процесс. Для Netlify лучше делать webhook-версию.
