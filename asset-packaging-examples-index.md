# Каталог примеров упаковки активов

## 1. Назначение каталога

Этот документ объединяет все подготовленные демонстрационные примеры упаковки активов в единую структуру и показывает, как использовать их для дальнейших материалов: PDF, блоков сайта и клиентских презентаций.

Важно зафиксировать рамки использования:
- это демонстрационные примеры упаковки;
- это не завершённые кейсы продажи;
- материалы не заявляют, что активы были проданы;
- материалы показывают, что получает собственник после анализа исходных данных и подготовки упаковки.

## 3. Единая структура упаковки

Во всех примерах используется единая логика, которая позволяет стандартизировать работу и результат для клиента:

1. **Входные данные** — что получено на старте (документы, фото, описание, ограничения).
2. **Карточка актива** — краткий профиль объекта/актива в рабочем и презентационном формате.
3. **Карта покупателей** — сегменты потенциальных покупателей и приоритеты контакта.
4. **Карта рисков** — ключевые барьеры сделки и способы их снижения.
5. **Документы** — перечень обязательных и желательных подтверждающих материалов.
6. **Фото / видео** — визуальный контент, который влияет на доверие и конверсию.
7. **До / после упаковки** — сравнение исходного и упакованного состояния материала.
8. **Коммерческое позиционирование** — как актив подаётся рынку и на какие выгоды делается акцент.
9. **Маршрут реализации** — поэтапный сценарий продажи/реализации актива.
10. **Переговорная позиция** — аргументация по цене, условиям и работе с возражениями.
11. **План на 7 дней** — ближайшие действия после упаковки для старта процесса.
12. **Будущий PDF** — как упаковать пример в наглядный клиентский документ.
13. **Будущий сайт-блок** — как показать суть примера на сайте без перегрузки деталями.
14. **Материалы для реального кейса** — что нужно собрать, чтобы перевести пример в подтверждённый кейс.

## 4. Что можно показывать клиенту

| Материал | Когда показывать | Что доказывает | Риск / ограничение |
|---|---|---|---|
| Пример упаковки производственной базы | На этапе первичного брифа с владельцем производственного актива | Показывает глубину анализа и структуру подготовки к реализации | Нельзя подавать как завершённую продажу без подтверждённых фактов |
| Пример упаковки склада / ангара | При обсуждении стратегии продажи складских площадей | Демонстрирует методику позиционирования и сегментации покупателей | Без факта сделки это только демонстрация подхода |
| Пример упаковки оборудования / спецтехники | При входящих запросах по реализации оборудования | Подтверждает, что движимые активы также упаковываются по системе | Нельзя заявлять достигнутые финансовые результаты без доказательств |
| Пример упаковки складских остатков / ТМЦ | При работе с собственниками неликвидов и избыточных запасов | Показывает логику структурирования и маршрута реализации ТМЦ | Нельзя обещать итоговую выручку, если она не подтверждена фактически |
| Общий PDF-пример | На встречах, где нужно быстро объяснить весь подход целиком | Даёт целостную картину услуги и формата результата | При избыточных обещаниях может восприниматься как «кейс с результатом» |
| Блок сайта «Как выглядит упаковка актива» | На этапе самостоятельного изучения услуги клиентом | Повышает доверие за счёт прозрачности процесса | Требует аккуратных формулировок, чтобы не создать ложных ожиданий |

## 5. Какие PDF нужно сделать

### 1) PDF: Производственная база
- **Назначение:** показать формат глубокой упаковки крупного промышленного актива.
- **Целевой клиент:** собственник производственной базы, имущественного комплекса, индустриальной площадки.
- **Рекомендуемая структура:** вводные → карточка актива → покупатели → риски → документы → до/после → маршрут реализации → план 7 дней.
- **CTA:** «Запросить первичный разбор производственной базы».

### 2) PDF: Склад / ангар
- **Назначение:** показать упаковку складской недвижимости с акцентом на ликвидность и профиль покупателя.
- **Целевой клиент:** собственник склада, ангара, логистической площадки.
- **Рекомендуемая структура:** вводные → параметры объекта → сегменты покупателей → риски и ограничения → коммерческая подача → шаги реализации.
- **CTA:** «Получить разбор склада/ангара за 48 часов».

### 3) PDF: Оборудование / спецтехника
- **Назначение:** показать методику упаковки движимого актива для ускорения переговоров.
- **Целевой клиент:** собственник парка техники, производственного оборудования, спецмашин.
- **Рекомендуемая структура:** профиль оборудования → техсостояние → комплект документов → сценарии реализации → возражения и ответы → план запуска.
- **CTA:** «Запросить упаковку оборудования под продажу».

### 4) PDF: Складские остатки / ТМЦ
- **Назначение:** показать как структурировать остатки и подготовить их к поэтапной реализации.
- **Целевой клиент:** собственник складских остатков, дистрибьютор, производственная компания с неликвидами.
- **Рекомендуемая структура:** структура остатков → сегментация по ликвидности → каналы сбыта → ценовой контур → риски → дорожная карта реализации.
- **CTA:** «Получить план реализации складских остатков».

### 5) Общий PDF: Как выглядит упаковка актива
- **Назначение:** дать универсальный обзор услуги в одном документе.
- **Целевой клиент:** любой собственник актива на этапе знакомства с подходом.
- **Рекомендуемая структура:** что входит в упаковку → примеры по типам активов → до/после → ожидаемый формат результата → как стартовать.
- **CTA:** «Получить демонстрационный разбор вашего актива».

## 7. Что нужно для реальных кейсов

Чтобы переводить демонстрационные примеры в реальные кейсы, до публикации нужно собрать подтверждающие материалы:
- фотографии объекта/актива;
- старые объявления или исходные публикации;
- улучшенные описания после упаковки;
- комплект документов (в допустимом для публикации объёме);
- реальные возражения покупателей;
- историю показов/осмотров;
- историю изменения цены;
- таймлайн переговоров;
- актуальный статус (в работе, в переговорах, реализовано);
- разрешение клиента на публикацию или корректную анонимизацию.

Принцип публикации:
- не публиковать вымышленные успешные кейсы;
- если сделка не завершена, материал подавать как **«пример разборки и упаковки актива»**.

6. **Этап 6:** собрать реальные подтверждающие материалы и перевести примеры в анонимизированные кейсы.

## 9. Validation rules

Обязательные правила валидации:
- не заявлять фейковые факты о завершённых продажах;
- не придумывать покупателей;
- не придумывать выручку;
- не придумывать конверсию;
- использовать демонстрационные формулировки, пока нет подтверждённых доказательств;
- анонимизировать данные клиентов;
- чётко разделять примеры и реальные кейсы.

Проверка для текущей задачи:
- создан только файл `asset-packaging-examples-index.md`;
- production-файлы не изменены;
- HTML/CSS/JS/functions не изменены;
- существующие markdown-файлы не изменены;
- нет фейковых заявлений о продаже;
- нет фейковых результатов по покупателям.
