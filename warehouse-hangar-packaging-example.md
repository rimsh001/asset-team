# Пример упаковки склада / ангара

## 1. Назначение документа
Этот демонстрационный документ показывает, как команда A&A Asset Team переводит разрозненные данные по складу или ангару в структурированный сделочный файл перед выводом на рынок. Цель — не «украсить объявление», а собрать рабочую модель актива для покупателя: параметры объекта, инженерия, логистика, риски, документы, коммерческое позиционирование и маршрут реализации.

Важно: это именно демонстрационный пример упаковки. Он не является завершённым кейсом продажи, не фиксирует факт сделки и не содержит результатов покупателя.

## 2. Что получает собственник на выходе
- карточку объекта в едином формате;
- диагностику причин зависания спроса;
- карту целевых покупателей по сценариям использования;
- карту рисков покупателя с приоритетами;
- чек-лист документов со статусами;
- фото-чеклист и требования к видео;
- коммерческое позиционирование «под покупателя»;
- переговорную позицию без хаотичного дисконта;
- маршрут реализации по этапам;
- план действий на первые 7 дней.

## 3. Входные данные от собственника

| Блок данных | Что запросить | Зачем нужно |
|---|---|---|
| Базовые параметры | Площадь объекта, высота потолков, нагрузка на пол | Проверка соответствия операционному сценарию покупателя |
| Логистика здания | Количество ворот, тип ворот, доки/рамповые решения | Оценка скорости и стоимости складской операции |
| Инженерия | Отопление, электричество (мощность), вода, канализация | Проверка готовности объекта к эксплуатации |
| Пожарная безопасность | Категория, системы пожаротушения/сигнализации, эвакуация | Снижение регуляторного и страхового риска |
| Транспортная доступность | Подъезд фур, ограничения по габаритам, зона манёвра | Проверка фактической логистической пригодности |
| Территория | Площадь и контур территории, ограждение, режим доступа | Оценка безопасности и операционного резерва |
| Текущая занятость | Вакантность/занятость, арендаторы (если есть), условия | Понимание сценария входа покупателя |
| Документы | Правоустанавливающие, технические, инженерные, договорные | Подготовка прозрачной сделки |
| Фото и видео | Фото критичных зон, видео подъезда/разворота/внутренних потоков | Снижение «слепых зон» до выезда на просмотр |
| История экспозиции | Где и как размещали, какие были отказы | Диагностика причин зависания |

## 4. Почему склад / ангар может зависнуть
Склад или ангар чаще всего зависает из-за сочетания факторов:
- описание «про квадратные метры» без операционного сценария;
- не подтверждены высота, нагрузка на пол, доки и ворота;
- не раскрыты отопление, электричество, вода и канализация;
- слабая или неполная пожарная часть;
- не показан реальный подъезд фур и манёвровая зона;
- не объяснена текущая занятость и логика работы с арендаторами;
- документы не сведены в единый понятный пакет;
- цена не обоснована параметрами и рисками;
- переговоры рано уходят в торг «на скидку».

## 5. Карточка объекта

| Параметр | Содержание (демонстрационный пример) |
|---|---|
| Тип актива | Склад / ангар для логистики, хранения или лёгкой производственной операции |
| Площадь | Зафиксирована и структурирована по функциональным зонам |
| Высота потолков | Подтверждена по рабочим зонам размещения |
| Нагрузка на пол | Зафиксированы норматив и фактическое состояние покрытия |
| Ворота и доки | Описаны типы ворот, число точек погрузки/разгрузки, доступность для фур |
| Отопление | Указан фактический режим работы в сезон |
| Электричество | Подтверждённая мощность и режим подключения |
| Вода и канализация | Наличие/отсутствие и влияние на сценарий эксплуатации |
| Пожарная безопасность | Состав систем и статус ключевых подтверждений |
| Подъезд фур и манёвр | Описаны маршрут въезда, узкие места, зона разворота |
| Территория | Контур, покрытие, доступ, ограждение, резерв под операцию |
| Текущая занятость | Полностью свободен / частично занят / занят арендаторами |
| Арендаторы (если есть) | Структура размещения и базовые условия перехода |
| Документы | Статус правоустанавливающих, технических и инженерных документов |
| Фото и видео | Набор подтверждений по обязательному чек-листу |

## 6. Карта целевых покупателей

| Сегмент | Сценарий интереса | Что проверяет первым |
|---|---|---|
| 3PL / логистический оператор | Распределительный или перевалочный узел | Ворота/доки, подъезд фур, манёвровая зона, режим доступа |
| Дистрибьютор | Региональный склад запаса и отгрузок | Площадь, высота, скорость обработки, территория |
| Производственная компания | Склад сырья/готовой продукции рядом с производством | Нагрузка на пол, инженерия, пожарная часть |
| e-commerce / fulfillment | Операция с высокой плотностью отбора и отгрузки | Потоки, доки, электричество, операционная гибкость |
| Сервисная компания | База хранения материалов и оборудования | Доступ, охрана, коммунальные мощности |
| Соседний бизнес | Расширение текущей логистики рядом | Юридическая чистота, скорость входа |
| Инвестор под аренду | Доходная модель через арендаторов | Ликвидность формата, риски, договорный контур |
| Девелопер / redevelopment | Перепрофилирование площадки | Ограничения, регламенты, потенциал развития |

## 7. Карта рисков покупателя

| Риск | Как видит покупатель | Что закрывает риск |
|---|---|---|
| Документы | Сделка может затянуться или сорваться | Полный реестр документов + план закрытия пробелов |
| Высота и площадь | Функция не помещается в реальный объём | Обмеры и поэтажные/зональные данные |
| Полы и нагрузка | Ограничение по технике и стеллажам | Подтверждение характеристик пола и дефектов |
| Отопление | Сезонная эксплуатация станет дорогой/невозможной | Техданные по системе и фактическому режиму |
| Пожарная безопасность | Регуляторные и страховые ограничения | Пакет подтверждений по ПБ |
| Подъезд фур | Срыв логистики из-за ограничений доступа | Фото/видео маршрута и габаритов |
| Маневровая зона | Потери времени на разворот и постановку под разгрузку | Схема движения и визуальное подтверждение |
| Скрытые вложения | Бюджет входа окажется выше ожиданий | Карта CAPEX по критичным зонам |
| Цена | Цена не бьётся с риском и форматом | Логика цены с корректировками |
| Срок сделки | Длинный цикл снижает интерес | Пошаговый таймлайн и ответственные |

## 8. Чек-лист документов
- правоустанавливающие документы на землю и строения;
- выписки и кадастровые сведения;
- техническая документация по объекту;
- данные по отоплению, электричеству, воде и канализации;
- документы по пожарной безопасности;
- сведения по текущей занятости;
- договоры аренды и условия перехода (если есть арендаторы);
- подтверждения по ограничениям/обременениям (при наличии);
- структурированная карточка объекта для переговоров.

## 9. Фото-чеклист

| Зона | Что снять | Зачем |
|---|---|---|
| Внешний контур | Фасады, въезд, ограждение, территория | Подтвердить класс и доступность актива |
| Ворота и доки | Каждые ворота, доклевеллеры, рампы, точки погрузки | Оценить скорость грузовой операции |
| Высота и объём | Общие планы внутри, отметки высоты, конструктив | Проверить вместимость под сценарий |
| Полы | Состояние покрытия, швы, дефекты, зоны нагрузки | Оценить риск дополнительных вложений |
| Инженерия | Щитовые, отопление, узлы воды/канализации | Снять сомнения по эксплуатационной готовности |
| Пожарная часть | Пожарные шкафы, датчики, пути эвакуации | Подтвердить базовую комплаенс-готовность |
| Подъезд фур | Маршрут въезда, узкие места, разворот | Показать реальную логистику на площадке |
| Манёвровая зона | Постановка транспорта под ворота/доки | Проверить операцию без «ручных» допущений |
| Территория | Покрытие, освещение, парковка, резервные зоны | Понять потенциал масштабирования |
| Видео | Короткие проходы: въезд, манёвр, внутренняя логистика | Дать непрерывную картину эксплуатации |

## 12. Коммерческое позиционирование
**Слабая формулировка:**  
«Продаётся склад / ангар. Площадь большая, есть ворота, возможен торг.»

**Сильная формулировка:**  
«Предлагается складской/ангарный актив как готовый операционный сценарий для логистики и хранения: подтверждённые параметры площади и высоты, рабочая нагрузка на пол, организованные ворота и зоны погрузки, базовая инженерия, а также проверенный маршрут подъезда фур и маневрирования на территории. Покупатель оценивает не просто квадратные метры, а предсказуемую модель запуска складской операции с понятными рисками, документами и сроками входа.»

## 13. Маршрут реализации
| Этап | Содержание |
|---|---|
| 1. Сбор данных | Приём исходной информации и первичная сверка фактов |
| 2. Диагностика | Поиск причин зависания и критичных пробелов |
| 3. Документы | Формирование реестра и закрытие приоритетных рисков |
| 4. Визуал | Съёмка по фото/видео-чеклисту |
| 5. Сегментация | Подбор целевых покупателей по сценариям |
| 6. Упаковка | Сборка deal-file в едином стандарте |
| 7. Вывод | Запуск в релевантные каналы и первичная квалификация |
| 8. Переговоры | Ведение диалога по структуре сделки и next step |

## 14. Переговорная позиция собственника
- Не раскрывать на первом касании нижнюю границу уступки.
- Подчёркивать подтверждённые параметры (высота, полы, ворота, логистика), а не общие слова.
- Переводить торг в обсуждение структуры сделки и состава передаваемых условий.
- На вопрос «какая минимальная цена?» использовать рабочий ответ:  
  «Корректно обсуждать цену после сверки сценария использования, состава передаваемых параметров и сроков входа. Мы открыты к предметному диалогу в рамках обоснованной модели.»

## 15. Решение для собственника на первые 7 дней

| День | Действие | Результат |
|---|---|---|
| День 1 | Сбор baseline по объекту и занятости | Зафиксирован состав актива и статус использования |
| День 2 | Проверка правовой базы | Понятен контур документов и пробелы |
| День 3 | Сверка инженерии и ПБ | Собран технический блок для покупателя |
| День 4 | Фото и видео по чек-листу | Убран основной визуальный риск |
| День 5 | Сегментация покупателей | Сформирована карта приоритетных сценариев |
| День 6 | Риск-матрица и ценовая рамка | Подготовлена переговорная логика |
| День 7 | Итоговый owner-summary | Зафиксирован маршрут следующего шага |

## 16. Как должен выглядеть клиентский PDF
Клиентский PDF — это аккуратный B2B deal-file:
- графитовая обложка;
- кремовые внутренние страницы;
- золотые акценты для ключевых блоков;
- компактные таблицы параметров, рисков и документов;
- отдельный блок логистики (фуры, доки, манёвр);
- отдельный блок инженерии и пожарной безопасности;
- статус-теги готовности материалов.

## 17. Что можно показать на сайте
Публичный блок без конфиденциальных деталей:
- «Как выглядит упаковка склада / ангара»;
- пример карточки объекта;
- пример buyer map;
- пример risk map;
- укороченный чек-лист документов;
- укороченный фото-чеклист;
- CTA: «Получить первичный разбор объекта».

## 18. Что нужно для реального кейса
Для подготовки реального кейса (только при согласовании публикации или анонимизации) нужны:
- исходное объявление;
- комплект фото и видео до/после;
- фактический перечень документов и статусы;
- журнал возражений покупателей;
- история показов и обратной связи;
- история корректировок цены;
- таймлайн переговорного процесса;
- подтверждение, какие данные можно публиковать.

## 19. Итог
Склад или ангар выводится на рынок эффективно, когда покупателю показывают не «просто площадь», а операционный сценарий: подтверждённые параметры объекта, логистику фур, инженерную и пожарную готовность, прозрачные документы, карту рисков и управляемый маршрут сделки. Этот демонстрационный шаблон нужен для стандартизации упаковки до запуска в рынок.
