# Пример упаковки производственной базы

## 1. Назначение документа
Этот демонстрационный документ показывает, как команда A&A Asset Team переводит разрозненные исходные данные от собственника производственной базы в структурированный сделочный файл. Цель — не «украсить объявление», а собрать понятную для покупателя модель актива: состав, документы, инженерия, логистика, риски, ценовая логика и маршрут реализации.

## 2. Что получает собственник на выходе
- карточка актива;
- диагностика причин зависания;
- карта целевых покупателей;
- карта рисков покупателя;
- чек-лист документов;
- фото-чеклист;
- логика цены;
- маршрут реализации;
- переговорная позиция;
- итоговое резюме.

## 3. Входные данные от собственника

| Блок данных | Что запросить | Зачем нужно |
|---|---|---|
| Объект | Адрес, тип базы, текущий формат использования, площадь территории | Зафиксировать предмет сделки и сценарии использования |
| Собственность | Кто собственник (юрлицо/физлицо), структура владения, ограничения по отчуждению | Проверить готовность к сделке и полномочия |
| Земля | Кадастровый номер, площадь, ВРИ, форма права (собственность/аренда), срок аренды | Оценить юридическую устойчивость и профиль покупателя |
| Здания | Перечень строений, площади, назначение, год постройки/реконструкции | Понять реальный состав актива и объем проверки |
| Коммуникации | Электричество (мощность), вода, канализация, газ, отопление, резервные источники | Подтвердить производственную пригодность |
| Доступ и логистика | Подъездные пути, покрытие, ограничения по транспорту, расстояния до узлов | Оценить операционную логистику покупателя |
| Состояние | Техническое состояние зданий и площадок, текущие дефекты, необходимость ремонта | Сформировать инвестиционные ожидания и аргументацию цены |
| Цена | Текущая цена, условия оплаты, допустимая структура сделки | Подготовить реалистичную стратегию переговоров |
| История продажи | Срок экспозиции, где размещали, какая реакция, причины отказов | Найти точки зависания и исправить позиционирование |
| Документы | Правоустанавливающие, технические, инженерные, градостроительные документы | Снизить юридические и технические риски покупателя |
| Фото/видео | Актуальные фото территории, зданий, коммуникаций, дефектов, подъездов | Сформировать доверие и сократить «слепые» возражения |

## 4. Диагностика: почему база может зависнуть
Производственная база обычно «зависает» не из-за одного фактора, а из-за связки слабых мест:
- неясный состав актива;
- слабые документы;
- неясные коммуникации;
- слабые фото;
- нет сценария покупателя;
- необъяснённая цена;
- риски покупателя не сняты;
- неверный канал продажи;
- переговоры скатываются в дисконт без обоснования.

## 5. Карточка актива

| Параметр | Содержание (пример демонстрационного заполнения) |
|---|---|
| Тип | Производственная база смешанного назначения |
| Регион | Промышленный кластер регионального центра |
| Земельный участок | Единый контур с возможностью поэтапного освоения |
| Здания | Цеха, складские и административно-бытовые помещения |
| Коммуникации | Базовый комплект инженерии для промышленной эксплуатации |
| Доступ | Круглогодичный подъезд грузового транспорта |
| Состояние | Рабочее, с зонами для плановой модернизации |
| Документы | Базовый пакет есть, часть инженерных подтверждений требует доукомплектования |
| Текущий статус | Эксплуатируется/частично эксплуатируется |
| Текущая цена | Определена собственником, требует защитимой ценовой логики |
| Главная задача | Упаковка в формат понятного покупателю операционного сценария |

**Сильные стороны:**
- сформированный промышленный контур;
- наличие зданий и базовой инженерии;
- возможность запуска в коротком горизонте после сделки.

**Ограничения:**
- неполнота части подтверждающих документов;
- возможные вложения в обновление отдельных зон;
- необходимость точного сегментирования покупателей.

## 6. Карта целевых покупателей

| Сегмент | Зачем им база | Что они будут проверять | Какой аргумент нужен |
|---|---|---|---|
| Производственная компания | Быстрый запуск/расширение мощностей | Мощности, техсостояние, ВРИ, сроки запуска | «Можно перейти к операционной фазе без длительного greenfield-цикла» |
| Логистический оператор | Узел складирования и распределения | Логистика, манёвренность, покрытие, доступ 24/7 | «База поддерживает интенсивную грузовую операцию» |
| Сервисная компания | Размещение ремонтной/технической базы | Зонирование, инженерия, сервисные площади | «Есть контур для производственно-сервисного сценария» |
| Соседний бизнес | Расширение рядом с текущей площадкой | Синергия с текущими мощностями, юридическая чистота | «Минимальные издержки интеграции в действующий периметр» |
| Инвестор | Покупка с последующей арендной/операционной моделью | Доходность, риски, ликвидность, CAPEX | «Понятная модель монетизации и карта рисков» |
| Девелопер / редевелопмент | Перепрофилирование территории | Градрегламент, ВРИ, ограничения, экономика проекта | «Площадка имеет потенциал сценарного развития» |

## 7. Карта рисков покупателя

| Риск | Как покупатель это видит | Какое подтверждение нужно | Приоритет |
|---|---|---|---|
| Документы | «Сделка может затянуться или сорваться» | Полный реестр документов и план закрытия пробелов | Высокий |
| Коммуникации | «Фактические мощности ниже заявленных» | Договоры, техусловия, акты, подтверждение лимитов | Высокий |
| Состояние зданий | «После покупки потребуются внеплановые вложения» | Актуальное техописание, ведомость дефектов, смета работ | Высокий |
| Использование земли | «Сценарий может не соответствовать ВРИ/ограничениям» | Выписки, ГПЗУ/регламентные данные, правовой комментарий | Высокий |
| Логистика | «Ограниченный доступ ухудшит операционную модель» | Фото/видео подъездов, схемы движения, замеры | Средний |
| Цена | «Цена не опирается на проверяемую экономику» | Ценовая логика, сравнительные ориентиры, модель корректировок | Высокий |
| Сроки сделки | «Слишком долгий цикл отвлекает ресурсы» | Пошаговый таймлайн и ответственные стороны | Средний |
| Скрытые инвестиции | «Реальный бюджет входа выше заявленного» | CAPEX-карта по этапам запуска | Высокий |

## 8. Чек-лист документов

**Базовые документы:**
- правоустанавливающие документы на землю и здания;
- выписки из реестров;
- кадастровые/технические сведения;
- документы по ограничениям/обременениям (при наличии).

**Инженерные документы:**
- договоры и условия подключения коммуникаций;
- сведения о доступной мощности и фактическом потреблении;
- схемы инженерных сетей и узлов учёта;
- данные о последних обследованиях/ремонтах.

**Переговорные документы:**
- карточка актива в едином формате;
- реестр рисков и закрывающих подтверждений;
- логика цены и допустимые параметры сделки;
- маршрут реализации и таймлайн следующего шага.

## 10. Коммерческое позиционирование

**Слабая формулировка:**
«Продаётся производственная база. Есть земля, здания, коммуникации. Возможен торг.»

**Сильная формулировка:**
«Предлагается производственная база как готовый операционный контур: земельный участок, функциональные здания и подтверждённая инженерная инфраструктура, позволяющие запускать производственный или сервисно-логистический сценарий в прогнозируемые сроки. Структура актива, документы, риски и этапы входа в объект собраны в единый сделочный файл, чтобы покупатель оценивал не “набор квадратных метров”, а рабочую модель бизнеса.»

## 11. Маршрут реализации

| Этап | Содержание |
|---|---|
| 1. Приём данных | Сбор исходной информации и первичная верификация |
| 2. Диагностика | Поиск причин зависания и узких мест |
| 3. Документы | Формирование реестра и закрытие критичных пробелов |
| 4. Фото и описание | Подготовка визуала и структурированного описания |
| 5. Карта покупателей | Выбор целевых сегментов и сценариев обращения |
| 6. Упаковка | Сборка единого deal-file для вывода на спрос |
| 7. Вывод на спрос | Запуск по релевантным каналам и форматам |
| 8. Квалификация покупателей | Фильтрация входящих запросов и проверка намерений |
| 9. Переговоры | Ведение диалога по структуре сделки, а не по хаотичному торгу |
| 10. Следующий шаг | Фиксация действия: просмотр, обмен документами, проект условий |

## 12. Переговорная позиция собственника
- **Что не раскрывать слишком рано:** нижнюю границу уступки, внутреннюю срочность продажи, неподтверждённые допущения по инженерии.
- **Что можно раскрывать на первом этапе:** состав актива, базовую документальную структуру, подтверждённые параметры по доступу и эксплуатации.
- **Профессиональный ответ на вопрос «какая минимальная цена?»:**
  «Минимальная цена корректно обсуждается после того, как мы сверили сценарий использования, структуру сделки и набор передаваемых параметров. Мы открыты к предметному диалогу в рамках обоснованной модели.»

## 13. Как должен выглядеть клиентский PDF
Клиентский PDF оформляется как аккуратный сделочный файл:
- графитовая обложка;
- кремовые внутренние страницы;
- золотые акценты для смысловых блоков;
- структурные таблицы по активу и рискам;
- чек-листы для документов и фото;
- риск-лейблы по приоритетам;
- единый «deal-file look», читаемый для собственника и покупателя.

Для подготовки реального кейса (с разрешением на публикацию или с анонимизацией) нужны:
- фотографии;
- старое объявление;
- улучшенное описание;
- список документов;
- возражения покупателей;
- история показов;
- история цены;
- таймлайн переговоров;
- статус на текущий момент;
- разрешение на публикацию или анонимизация данных.

## 16. Итог
Производственная база не должна выводиться на рынок как «земля + здания». Для управляемой реализации её необходимо упаковать в операционный сценарий: с прозрачными рисками, подтверждёнными документами, логистикой, инженерией, ценовой логикой и понятным таймлайном запуска сделки.
