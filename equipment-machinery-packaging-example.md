# Пример упаковки оборудования / спецтехники

## 1. Назначение документа
Этот документ — демонстрационный пример упаковки оборудования / спецтехники перед выводом на рынок. Цель примера — показать, как собственник переводит актив из «сырого объявления» в управляемую сделку с понятной логикой для покупателя. Это не завершённый кейс продажи и не отчёт о закрытой сделке, а рабочий шаблон подготовки к реализации.

## 2. Что получает собственник на выходе
Собственник получает на выходе:
- карточка актива;
- карта целевых покупателей;
- карта рисков покупателя;
- чек-лист документов;
- фото- и видео-чеклист;
- дефект-лист;
- ведомость комплектности;
- логистический контур;
- переговорная позиция;
- план на первые 7 дней.

## 3. Входные данные от собственника
| Блок данных | Что запросить | Зачем нужно |
|---|---|---|

| Техническое состояние | Текущее состояние по узлам | Понимание ремонтных рисков покупателя |
| Эксплуатация и сервис | История обслуживания | Подтверждение дисциплины эксплуатации |
| Ремонтный контур | Ремонты / дефекты | Формирование честной картины состояния |
| Состав поставки | Комплектность | Снижение споров при осмотре и передаче |
| Правовой блок | Документы и право собственности | Юридическая чистота и возможность оформления |

| Погрузочный контур | Погрузка | Определение ответственных и техники |
| Транспортный контур | Перевозка | Сроки, стоимость и маршрут доставки |
| География | Местонахождение | Логистика, налоги, доступность площадки |
| Ценовой ориентир | Цена | Стартовая экономика переговоров |
| Тайминг | Срочность продажи | Выбор стратегии вывода и глубины торга |

## 4. Почему оборудование / спецтехника может зависнуть
Оборудование и спецтехника часто «зависают» на рынке из-за повторяющихся блокеров:
- состояние описано общими словами;
- нет видео запуска;
- неясная комплектность;
- нет дефект-листа;
- документы разрознены;
- не просчитан демонтаж;
- не просчитана перевозка;
- цена не объяснена;
- покупатели задают одни и те же вопросы;
- торг начинается слишком рано.

## 5. Карточка актива
| Параметр | Содержание |
|---|---|
| Тип актива | Фрезерный обрабатывающий центр (демонстрационный пример) |
| Марка / модель | Указывается собственником |
| Год выпуска | Указывается собственником |
| Наработка | Указывается собственником |
| Серийный номер | Указывается собственником |
| Комплектность | Базовая поставка + фактически передаваемые позиции |
| Техническое состояние | Рабочее, с фиксацией узлов по чек-листу осмотра |
| Обслуживание | Подтверждается сервисными документами |
| Ремонт / дефекты | Отражаются в дефект-листе без сокрытия |
| Документы | Право собственности, паспорта, закрывающие документы |
| Регистрация | По применимости для типа актива |
| Локация | Адрес площадки и режим доступа |
| Осмотр | По согласованному окну времени |
| Запуск / тест | По заранее утверждённому сценарию |
| Демонтаж | Варианты: силами продавца / покупателя |
| Погрузка | Ответственные лица и техника погрузки |
| Перевозка | Возможные форматы и ограничения маршрута |
| Цена | Стартовая цена с логикой обоснования |
| Главная задача | Подготовить актив к прогнозируемым переговорам |

**Сильные стороны**
- Понятная структура данных по состоянию и комплектности.
- Прозрачный документальный контур для предварительной проверки.
- Подготовленный сценарий осмотра, теста и логистики.

**Ограничения**
- Финальная оценка покупателя зависит от результатов очного осмотра.
- Стоимость демонтажа и перевозки меняется от региона и сроков.
- Некоторые параметры требуют подтверждения исходными документами собственника.

## 6. Карта целевых покупателей
| Сегмент | Зачем ему актив | Что будет проверять | Какой аргумент нужен |
|---|---|---|---|
| Профильное производство | Быстро закрыть производственную задачу | Ресурс узлов, точность, сервисную историю | Готовность к запуску и предсказуемая эксплуатация |
| Строительная компания | Использовать в текущих проектах | Надёжность, ремонтопригодность, сроки ввода | Минимизация простоев на объекте |
| Складской / логистический оператор | Увеличить операционную мощность | Нагрузочные режимы, износ, безопасность | Экономика владения и режим обслуживания |
| Сервисная компания | Выполнять подрядные работы | Универсальность и доступность запчастей | Быстрая окупаемость под текущий портфель заказов |
| Дилер / перекупщик | Перепродажа с маржой | Ликвидность модели и глубину дисконта | Потенциал перепродажи при прозрачных рисках |
| Арендная компания | Сдать в аренду без длинной подготовки | Надёжность и стоимость подготовки к аренде | Сценарий быстрого ввода в аренду |
| Компания после расширения парка | Унифицировать или дополнить парк | Совместимость с текущим парком | Снижение издержек на интеграцию |
| Покупатель на запчасти | Разобрать на узлы | Состав и состояние ключевых компонентов | Подтверждённая комплектность по узлам |
| Экспортный покупатель | Вывезти и запустить в другой юрисдикции | Документы, маркировки, логистику вывоза | Подготовленность к трансграничной логистике |

## 7. Карта рисков покупателя
| Риск | Как видит покупатель | Что нужно подготовить | Приоритет |
|---|---|---|---|
| Техническое состояние | «Не понимаю фактический ресурс» | Диагностический лист по ключевым узлам | Высокий |
| Скрытые дефекты | «После покупки могут всплыть проблемы» | Честный дефект-лист + видеофиксация | Высокий |
| Документы | «Могут быть проблемы с оформлением» | Пакет правоустанавливающих и техдокументов | Высокий |
| Невозможность тестового запуска | «Покупаю без проверки» | Сценарий теста и окно для демонстрации | Высокий |
| Демонтаж | «Не понимаю сложность и бюджет» | План демонтажа с ответственными | Средний |
| Погрузка | «Риск повреждения и простоев» | Регламент погрузки и техника | Средний |
| Перевозка | «Неясен срок и итоговая стоимость» | Предварительный логистический расчёт | Средний |
| Комплектность | «Могут не передать часть узлов» | Ведомость комплектности с фото | Высокий |
| Ликвидность модели | «Смогу ли потом продать» | Сравнение с рыночными аналогами | Средний |
| Цена | «Цена выглядит завышенной» | Ценовая логика с факторами состояния | Высокий |
| Срок сделки | «Процесс может затянуться» | Пошаговый таймлайн сделки | Средний |

## 8. Чек-лист документов
Документальный пакет для упаковки:
- документы права собственности;
- договоры приобретения;
- закрывающие документы;
- паспорта;
- серийные данные;
- регистрационные документы;
- сервисная история;
- акты ремонта;
- дефектовка;
- фотофиксация шильдиков;
- полномочия подписанта;
- условия доступа на площадку.

## 9. Фото- и видео-чеклист
Фото- и видеоматериалы должны покрывать:
- общий вид;
- узлы;
- рабочий орган;
- кабина / панель;
- шильдик;
- серийный номер;
- зоны износа;
- дефекты;
- комплект поставки;
- навесное оборудование;
- видео холодного запуска;
- видео работы;
- видео погрузки, если возможно.

## 10. До / после упаковки
| Элемент | До упаковки | После упаковки |
|---|---|---|
| Описание | Общие формулировки | Структурированная карточка актива |
| Фото / видео | Случайный набор | Чек-лист фото + сценарные видео |
| Документы | Частично собраны | Пакет с понятной структурой |
| Техническое состояние | «Рабочее» без детализации | Узловая расшифровка + дефект-лист |
| Комплектность | Не зафиксирована | Ведомость комплектности |
| Логистика | Не просчитана | Контур демонтажа, погрузки и перевозки |
| Покупатель | Не сегментирован | Карта целевых сегментов |
| Риски | Не управляются | Карта рисков и ответы на возражения |

| Следующий шаг | Не определён | План действий на 7 дней |

## 11. Что было на входе / что стало на выходе
| Входные данные | Проблема | Что делаем | Выходной материал |
|---|---|---|---|
| Старое объявление | Нет структуры и фокуса | Пересобираем под сценарий покупателя | Новая карточка актива |
| Фото без структуры | Невозможно оценить состояние | Снимаем по чек-листу | Структурированный фотопакет |
| Нет видео запуска | Высокий риск недоверия | Пишем холодный запуск и работу | Видео подтверждения работоспособности |
| Неясная комплектность | Споры на этапе сделки | Формируем ведомость | Подписываемая комплектность |
| Нет дефект-листа | Скрытые риски | Фиксируем дефекты заранее | Прозрачный дефект-лист |
| Документы разрознены | Затяжная проверка | Собираем в единый пакет | Чек-лист и папка документов |
| Не просчитана логистика | Неясна итоговая экономика | Считаем демонтаж/погрузку/перевозку | Логистический контур |
| Цена без объяснения | Агрессивный торг с первого контакта | Формируем ценовую логику | Обоснованная переговорная цена |
| Покупатели задают один и тот же вопрос | Потеря времени команды | Готовим Q&A и позицию | Скрипт квалификации и ответов |

## 12. Коммерческое позиционирование
Слабая формулировка:

> «Продаётся оборудование / спецтехника. В рабочем состоянии, возможен торг.»

Сильная buyer-facing формулировка:

Актив подготовлен к предметному рассмотрению: зафиксированы текущее состояние по ключевым узлам, фактическая комплектность и документальный пакет. Осмотр проводится на площадке по согласованному окну; доступен тестовый запуск по утверждённому сценарию. Предварительно проработаны демонтаж, погрузка и варианты перевозки, чтобы покупатель заранее понимал полную экономику владения. Материал подан через сценарий использования у покупателя, а не через общие обещания.

## 14. Переговорная позиция собственника
Что нельзя раскрывать слишком рано:
- нижнюю границу цены до квалификации покупателя;
- внутреннюю срочность, если она ослабляет позицию;
- альтернативные предложения без подтверждённого статуса.

Что можно раскрывать на первом этапе:
- базовые параметры актива и локацию;
- подтверждённую комплектность;
- формат осмотра и тестового запуска;
- перечень документов, доступных для проверки.

Ответ на «какая минимальная цена?»
- «Минимальная цена обсуждается после осмотра и понимания состава поставки. Сейчас ориентир — цена из карточки с обоснованием по состоянию и комплектности.»

Ответ на «можно посмотреть в работе?»
- «Да, возможен просмотр в работе по согласованному времени и сценарию теста, чтобы вы увидели критичные узлы под нагрузкой.»

Ответ на «кто оплачивает демонтаж и доставку?»
- «Вариант зависит от структуры сделки: можем считать отдельно силами покупателя или включать часть работ в общий контур при согласовании условий.»

Ответ на «какая гарантия, что оно исправно?»
- «Мы не даём абстрактных обещаний: предоставляем фактуру — дефект-лист, материалы запуска и пакет обслуживания, чтобы решение принималось на проверяемых данных.»

## 19. Итог
Оборудование и спецтехника не должны продаваться формулой «рабочее состояние и возможен торг». Для управляемой реализации актив необходимо упаковывать через фактическое состояние, комплектность, документы, сценарий осмотра, логистику, карту рисков, ценовую логику и сценарий применения у конкретного покупателя.
