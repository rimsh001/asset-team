## 9. Решение для собственника на первые 7 дней

Фокус первой недели — собрать фактуру по активу, подтвердить текущее состояние, выровнять документы и подготовить переговорную позицию без обещаний, которые нельзя подтвердить проверкой.

## 10. Коммерческое позиционирование

Слабая формулировка, которая обесценивает актив:

> «Продаётся оборудование / спецтехника. В рабочем состоянии, возможен торг.»

Сильное buyer-facing позиционирование:

- **Состояние:** зафиксированы текущее техническое состояние, следы износа, критичные и некритичные дефекты.
- **Комплектность:** описан фактический состав поставки (базовый узел, навесное, расходники, ЗИП, недостающие позиции).
- **Документы:** собран перечень документов по владению, обслуживанию, ремонту и идентификации единицы.
- **Осмотр:** для покупателя предусмотрен регламент очного/дистанционного осмотра по чек-листу.
- **Тестовый запуск:** определён сценарий демонстрации работоспособности и фиксируемые параметры.
- **Логистика:** заранее просчитаны ограничения по погрузке, маршруту, срокам и сопутствующим затратам.
- **Сценарий использования покупателем:** показано, где и как актив может быть введён в работу после приобретения.

## 11. Маршрут реализации

| Этап | Действие | Результат |
|---|---|---|
| 1. Приём данных | Сбор исходной информации от собственника, фотоархива, документов, текущих вводных по цене | База фактов для диагностики и упаковки |
| 2. Диагностика | Технический и коммерческий разбор состояния и ликвидности | Карта сильных сторон, рисков и ограничений |
| 3. Документы | Верификация и структурирование комплекта документов | Прозрачный документальный контур для покупателя |
| 4. Фото и видео | Подготовка доказательного визуального пакета, включая запуск | Материалы для дистанционной квалификации спроса |
| 5. Карта покупателей | Сегментация целевых типов покупателей и их критериев | Приоритетный список релевантного спроса |
| 6. Упаковка | Сборка карточки актива, дефектной карты, сценария переговоров | Готовый коммерческий пакет для выхода на рынок |
| 7. Вывод на спрос | Публикация и адресная рассылка по целевым каналам | Первичный поток обращений |
| 8. Квалификация покупателей | Отсев нерелевантных запросов, фиксация серьёзности намерений | Короткий список потенциальных сделок |
| 9. Осмотр / тест | Организация осмотра и/или тестового запуска по сценарию | Подтверждение интереса на фактах |
| 10. Переговоры | Согласование цены, условий передачи, сроков и ответственности сторон | Структурированная переговорная позиция |
| 11. Следующий шаг | Фиксация конкретного next step (документы, задаток, дата) | Управляемое продвижение к сделке |

## 12. Как должен выглядеть клиентский PDF

Структура клиентского PDF:

1. Обложка
2. Executive summary
3. Карточка актива
4. Состояние и комплектность
5. Документы
6. Фото / видео
7. Карта покупателей
8. Карта рисков
9. Логистика
10. Переговорная позиция
11. План на 7 дней
12. Следующие действия

Дизайн-принципы:

- графитовая обложка (graphite cover);
- кремовые внутренние страницы (cream pages);
- золотые акценты (gold accents);
- структурные таблицы;
- чек-листы для проверки готовности;
- риск-лейблы по блокам;
- визуальная подача в формате deal-file.

## 14. Что нужно для реального кейса

Для перехода от демонстрационного примера к реальному кейсу нужны:

- фото;
- видео запуска;
- старое объявление;
- улучшенное описание;
- список документов;
- список дефектов;
- возражения покупателей;
- история осмотров;
- история цены;
- таймлайн переговоров;
- заметки по логистике;
- текущий статус;
- разрешение на публикацию или анонимизация.

## 15. Итог

Оборудование и спецтехника не должны продаваться формулой «в рабочем состоянии, возможен торг». Для управляемой реализации актив нужно упаковывать через подтверждённые состояние и комплектность, документы, сценарий осмотра и теста, логистику, карту рисков, ценовую логику и сценарий использования покупателем.
